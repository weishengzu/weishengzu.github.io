# 简述
近期开发项目的过程中遇到一个需求，**需要以特定的格式规范打印程序的运行日志**，具体的格式可能如下：

| timestamp |  PID  | TID    |   sys   | log-level | log-msg |
| --------- |------ |------- |---------|---------- |---------|
|  时间戳   | 进程id | 线程id | 运行环境 | 日志级别  | 日志内容 |
  
这么做的目的是为了能够统一公司各项目日志输出格式，这样项目引用其他小伙伴开发的库时，大家都以能相同的格式输出日志信息，方便大家调试与查bug，毕竟如果大家都按自己的方式来打印日志信息，那么日志看起来会非常没有规则，那真是谁看谁闹心😂


工作中有使用过google glog，非常强大，但是貌似自定义日志输出格式方面不太灵活，于是便又上网查了一下c++的日志框架，挺多的比如，glog、log4xx、spdlog等等。再上github上一搜，发现spdlog是目前最受欢迎的c++日志库，使用及其简单，各方面也满足我的需求，果断入坑。

## 简介
spdlog 是一个快速的 C++ 日志库，兼容 C++11，只包含头文件，是一个head-only库，这点很实用，不用处理c++麻烦的库依赖问题，集成简单，详细的使用方法推荐查看[官方文档](https://github.com/gabime/spdlog/wiki)


## 特性
大多数能想到的日志库特性它都支持，如下：
* 非常快
* 只包含头文件
* 无需依赖第三方库
* 支持跨平台 - Linux / Windows on 32/64 bits
* 支持多线程
* 支持异步模式(可选)
* 支持控制台日志输出(支持颜色)
* 支持日志输出级别
* 可自定义日志格式，功能丰富的格式，使用优秀的[fmt](https://github.com/fmtlib/fmt)库
>{fmt}是一个现代 C++ 字符串格式化开源库，为 C stdio 和 C++ iostreams 提供了一种快速安全的替代方案

## 基础概念
### sinks(日志记录器槽)
sink是实际将日志写入其目标的对象，每个sink应该只负责单个目标（例如文件、控制台、数据库）的日志写入操作，每一个sink也有自己的formatter对象设置日志的输出格式。sink具有 **_mt**（多线程multi threaded）或 **_st**（单线程single threaded）后缀来指示是否线程安全。sink也支持自定义，可查看[官方资料](https://spdlog.docsforge.com/v1.x/4.sinks/#available-sinks)。

**常用的sink**
* rotating_file_sink:当文件达到最大容量时自动创建新的日志文件，也称滚动日志或循环日志
* daily_file_sink:每天都创建新的日志文件，可以指定时间
* simple_file_sink:简单的日志文件，没有任何限制
* stdout_sink/stderr_sink: 输出到控制台的sink
* ostream_sink:写入到std::ostringstream
* null_sink:不会记录日志，直接删除
* syslog_sink:发送日志到syslog
* systemd_sink:发送日志到systemd
* dist_sink:分布式日志到所有的其他sink上
* msvc_sink:windows debug sink
* dup_filter_sink:移除重复的message的sink

### logger(日志记录器)
被顶层调用来输出日志的类。一个logger对象中存储有多个sink，每次调用log时(如果日志级别正确)，logger会在每个sink上调用sink(log_msg) 函数进行输出。与自带的sink对应，spdlog也自带了几种logger分别对应sink，logger也区分 **_mt** 或 **_st** 版本。

**使用工厂函数创建记录器**
```c++
//Create and return a shared_ptr to a multithreaded console logger.
#include "spdlog/sinks/stdout_color_sinks.h"
auto console = spdlog::stdout_color_mt("some_unique_name");
```
这将创建一个控制台记录器，在 spdlog 的全局注册表中将其注册为“some_unique_name”作为其 id，并将其作为 shared_ptr 返回，可以使用`spdlog::get("logger_name")`返回共享指针，从任何地方访问记录器。

**创建具有多个接收器的记录器**
```c++
std::vector<spdlog::sink_ptr> sinks;
sinks.push_back(std::make_shared<spdlog::sinks::stdout_sink_st>());
sinks.push_back(std::make_shared<spdlog::sinks::daily_file_sink_st>("logfile", 23, 59));
auto combined_logger = std::make_shared<spdlog::logger>("name", begin(sinks), end(sinks));
//register it if you need to access it globally
spdlog::register_logger(combined_logger);
```

spdlog也支持创建旋转、异步等logger，可查看[官方资料](https://spdlog.docsforge.com/v1.x/2.creating-loggers/)

>st/mt： 对象版本，spdlog中的logger对象和sink对象都有两种版本，一种是以st结尾的单线程版本，以及以mt结尾的多线程版本
>* st：单线程版本，不用加锁，效率更高
>* 多线程版本，用于多线程程序是线程安全的

#自定义格式
每个记录器的接收器都有一个格式化程序，用于将消息格式化到其目的地。
spdlog 的默认日志记录格式为：
`[2014-10-31 23:46:59.678] [my_loggername] [info] Some message`

有两种方法可以自定义记录器的格式
* 设置模式字符串（推荐）：
`set_pattern(pattern_string);`
* 或者继承spdlog::custom_flag_formatter类并实现`clone()`和`format(...)`抽象方法来自定义格式，并调用：
`set_formatter(std::make_unique<my_custom_formatter>());`([官方示例](https://github.com/gabime/spdlog/wiki/3.-Custom-formatting))

### 自定义格式set_pattern(..)
格式可以全局应用于**所有注册**的记录器，调用：
`spdlog::set_pattern("*** [%H:%M:%S %z] [thread %t] %v ***");`
或特定的记录器对象：
`some_logger->set_pattern(">>>>>>>>> %H:%M:%S %z %v <<<<<<<<<");`
或特定的接收器对象：
`some_logger->sinks()[0]->set_pattern(">>>>>>>>> %H:%M:%S %z %v <<<<<<<<<");`
`some_logger->sinks()[1]->set_pattern("..");`

### 格式对齐
每个模式标志都可以通过预先添加一个宽度编号（最多 64 个）来对齐。
使用`-`(左对齐) 或`=`(居中对齐) 控制对齐边：
| **对齐** | **意义** | **例子** | **结果** |
| --- | --- | --- | --- |
| `%<width><flag>` | 右对齐 | `%8l` | "    info" |
| `%-<width><flag>` | 左对齐 | `%-8l` | "info    " |
| `%=<width><flag>` | 居中对齐 | `%=8l` | "  info  " |

`!`如果结果的大小超过指定的宽度，可以选择添加以截断结果：
| **对齐** | **意义** | **例子** | **结果** |
| --- | --- | --- | --- |
| `%<width>!<flag>` | 右对齐或截断 | `%3!l` | "inf" |
| `%-<width>!<flag>` | 左对齐或截断 | `%-2!l` | "in" |
| `%=<width>!<flag>` | 居中对齐或截断 | `%=1!l` | "i" |

> **注意：** 要截断函数名称，请使用“！！”。例如%10!!将函数名称限制为 10 个字符。

## 日志刷新策略
默认情况下，spdlog会在它认为合适时刷新，输出到终端的日志应该是立即就刷新了，但输出到文件的日志会在程序正常关闭的时候才刷写到日志文件中，也就是说如果程序没有正常关闭，比如卡死或异常退出了，那么日志就会丢失。我们可以通过以下几种方式来修改刷新策略。

### 手动刷新
使用`logger->flush()`函数立即刷新，如果要确保每条日志都立即刷新，那么就在输出一条日之后立即调用该函数，例如：
```c++
console_logger->info(log);
console_logger->flush();
```

### 基于严重性的刷新
可以设置触发自动刷新的最低日志级别，每当记录的日志等级高于设置的最低等级时将会自动刷新，例如：
```c++
my_logger->flush_on(spdlog::level::err); //每当记录错误或更严重的消息时，这将触发刷新
```

### 定时刷新
设置刷新间隔时间，每隔一定时间就刷新一次，例如每 5s 刷新一次：
```c++
spdlog::flush_every(std::chrono::seconds(5));
```

## 自定义封装日志帮助类
有了以上的基本知识便可以封装一个满足我需求的帮助类了，这个类有以下几点基本要求：
* 能够将日志同时输出到控制台与指定的文件中
* 输出的日志格式统一直观，能够直接定位到打印日志的源函数
* 有方便的日志宏例如`LOG_INFO("hello word x={0}", 1)`打印日志
* 应用程序启动后初始化一次便可以全局使用
基于这些基本要求，封装了如下的帮助类：
```c++
/*
 * @Author: shengzu@wei
 * @Date: 2022-03-31 15:03:11
 * @LastEditTime: 2022-04-30 16:27:55
 * @LastEditors: shengzu@wei
 * @Description: logger_help.h
 * spdlog的简单封装，实现同时输出至控制台与文件中,cpp文件包含此头文件后即可使用输出宏打印日志
 * 日志输出格式为：时间戳 进程id 线程id 运行环境 包名 日志级别 源文件名 行号
 * 日志内容 e.g. 2022.04.01-14:51:35.782 32992-7100 Windows core.sdk [info] [core.cpp:47] hello word 
 * 日志文件命名: 运行环境_日期_包名.log 且为循环日志每个日志最大10m 
 * 日志文件输出e.g. 2022-04-01_core_sdk.log 2022-04-01_core_sdk.1.log 2022-04-01_core_sdk.2.log
 * 官方文档：https://github.com/gabime/spdlog/wiki
 */

#pragma once

#include <spdlog/async.h>
#include <spdlog/sinks/rotating_file_sink.h>
#include <spdlog/sinks/stdout_color_sinks.h>
#include <spdlog/spdlog.h>

#include <filesystem>
#include <iostream>
#include <locale>
#include <string>

// use fmt lib, e.g. LOG_WARN("warn log, {0}, {0}, {1}", 0, 1);
#define LOG_TRACE(msg, ...)                                                  \
  do {                                                                       \
    auto deault_log = spdlog::default_logger_raw();                          \
    if (nullptr == deault_log) break;                                        \
    deault_log->log(spdlog::source_loc{__FILE__, __LINE__, SPDLOG_FUNCTION}, \
                    spdlog::level::trace, msg, ##__VA_ARGS__);               \
  } while (0)
#define LOG_DEBUG(msg, ...)                                                  \
  do {                                                                       \
    auto deault_log = spdlog::default_logger_raw();                          \
    if (nullptr == deault_log) break;                                        \
    deault_log->log(spdlog::source_loc{__FILE__, __LINE__, SPDLOG_FUNCTION}, \
                    spdlog::level::debug, msg, ##__VA_ARGS__);               \
  } while (0)
#define LOG_INFO(msg, ...)                                                   \
  do {                                                                       \
    auto deault_log = spdlog::default_logger_raw();                          \
    if (nullptr == deault_log) break;                                        \
    deault_log->log(spdlog::source_loc{__FILE__, __LINE__, SPDLOG_FUNCTION}, \
                    spdlog::level::info, msg, ##__VA_ARGS__);                \
  } while (0)
#define LOG_WARN(msg, ...)                                                   \
  do {                                                                       \
    auto deault_log = spdlog::default_logger_raw();                          \
    if (nullptr == deault_log) break;                                        \
    deault_log->log(spdlog::source_loc{__FILE__, __LINE__, SPDLOG_FUNCTION}, \
                    spdlog::level::warn, msg, ##__VA_ARGS__);                \
  } while (0)
#define LOG_ERROR(msg, ...)                                                  \
  do {                                                                       \
    auto deault_log = spdlog::default_logger_raw();                          \
    if (nullptr == deault_log) break;                                        \
    deault_log->log(spdlog::source_loc{__FILE__, __LINE__, SPDLOG_FUNCTION}, \
                    spdlog::level::err, msg, ##__VA_ARGS__);                 \
  } while (0)
#define LOG_FATAL(msg, ...)                                                  \
  do {                                                                       \
    auto deault_log = spdlog::default_logger_raw();                          \
    if (nullptr == deault_log) break;                                        \
    deault_log->log(spdlog::source_loc{__FILE__, __LINE__, SPDLOG_FUNCTION}, \
                    spdlog::level::critical, msg, ##__VA_ARGS__);            \
  } while (0)

namespace fs = std::filesystem;  // c++17以上支持

class LoggerHelp final {
 public:
  static LoggerHelp& GetInstance() {
    static LoggerHelp logger;  // C++11 以上线程安全
    return logger;
  }

  bool InitLog(std::string log_file_path = "", std::string sys = "",
               std::string package = "") {
    log_file_path.empty()
        ? log_file_path_ = fs::current_path().u8string().append("/temp.log")
        : log_file_path_ = log_file_path;
    sys.empty() ? sys_name_ = "UnknownSys" : sys_name_ = sys;
    package.empty() ? package_name_ = "UnknownPackage"
                    : package_name_ = package;

    // check log path and try to create log directory
    fs::path log_path = fs::u8path(log_file_path_);
    fs::path log_dir = log_path.parent_path();
    fs::path log_name = log_path.filename();
    try {
      if (!fs::exists(log_path)) {
        fs::create_directories(log_dir);
      }
    } catch (const fs::filesystem_error& ex) {
      std::cout << "Log file create failed: " << ex.what()
                << " file_name is: " << log_file_path_ << std::endl;
    }
    try {
      // initialize spdlog
      constexpr std::size_t log_buffer_size = 32 * 1024;  // 32kb
      spdlog::init_thread_pool(log_buffer_size,
                               std::thread::hardware_concurrency());
      std::vector<spdlog::sink_ptr> sinks;

      std::string date = GetCurrentDate();
      std::string log_file =
          log_dir.string() + "/" + date + "_" + log_name.string();
      auto file_sink = std::make_shared<spdlog::sinks::rotating_file_sink_mt>(
          log_file, 1024 * 1024 * 5, 10);
      auto console_sink =
          std::make_shared<spdlog::sinks::stdout_color_sink_mt>();

      sinks.push_back(file_sink);
      sinks.push_back(console_sink);

      spdlog::set_default_logger(
          std::make_shared<spdlog::logger>("", sinks.begin(), sinks.end()));
      spdlog::set_pattern(
          "%Y.%m.%d-%H:%M:%S.%e %P-%t " + sys_name_ + " " + package_name_ +
          " %^[%l]%$ [%s:%#] " +
          "%v");  //设置日志输出格式,参考:https://github.com/gabime/spdlog/wiki/3.-Custom-formatting#customizing-format-using-set_pattern
      spdlog::flush_on(spdlog::level::trace);
      spdlog::set_level(spdlog::level::trace);

    } catch (const spdlog::spdlog_ex& ex) {
      std::cout << "Log init failed: " << ex.what()
                << " file_name is: " << log_file_path_ << std::endl;
      return false;
    }
    return true;
  }

  void ShutDown() { spdlog::shutdown(); }

  std::string GetFilePath() { return log_file_path_; }

 private:
  inline std::string GetCurrentDate() {
    std::time_t timep;
    std::time(&timep);
    char tmp[64];
    strftime(tmp, sizeof(tmp), "%Y-%m-%d", std::localtime(&timep));
    return tmp;
  }

 private:
  LoggerHelp() = default;
  ~LoggerHelp() = default;

  LoggerHelp(const LoggerHelp&) = delete;
  void operator=(const LoggerHelp&) = delete;

 private:
  std::atomic_bool is_inited_{false};
  std::string log_file_path_{"UNKNOWN"};
  std::string sys_name_{""};
  std::string package_name_{""};
};
```
使用方法如下：
```c++
#include "logger_help.h"

int main()
{
  LoggerHelp::GetInstance().InitLog("D:/test.log", "Windows", "demo.sdk");
  LOG_DEBUG("hello word!");
  LOG_INFO("Welcome to spdlog version {0}.{1}.{2}!", 1, 1, 0);
  ...
  LoggerHelp::GetInstance().ShutDown();
  return 0;
}

//输出如下:
2022.04.30-17:46:18.535 24836-25268 Windows demo.sdk [debug] [main.cpp:5] hello word!
2022.04.30-17:46:18.535 24836-24840 Windows demo.sdk [info] [main.cpp:6] Welcome to spdlog version 1.1.0!
```
